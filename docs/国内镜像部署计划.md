# 新游方向探索 - 国内镜像部署计划

## 一、背景与目标

### 1.1 现状分析

当前项目「新游方向探索」使用以下技术架构：

| 组件 | 技术方案 | 部署位置 | 备注 |
|------|---------|---------|------|
| 前端页面 | 原生 HTML + JS | Vercel | 海外访问尚可 |
| 文档编辑器 | CodeMirror 6 | - | 同上 |
| AI 对话面板 | 流式 SSE | Vercel Serverless | 海外访问慢 |
| 数据库 | Supabase (PostgreSQL) | 云端托管 | 国内外均可访问 |
| AI API | OpenRouter | 云端 API | 无地域限制 |

**核心问题**：Vercel 在海外部署，国内访问速度不理想，尤其是 AI 对话接口延迟较高。

### 1.2 目标

搭建一套**国内可快速访问的镜像站点**，实现：

- 海外用户 → 继续使用现有 Vercel 地址
- 国内用户 → 使用新的 Cloudflare Workers 国内节点
- 两套系统共享同一个 Supabase 数据库，内容完全同步
- AI 对话功能在国内节点也能正常使用

### 3.2 需要修改的部分

#### (1) `js/ai-panel.js` — API 地址适配

当前代码中的 AI 接口地址：

```javascript

const API_URL  = '/api/ai-chat';
```

**修改方案**：国内版使用相对路径，与海外版保持一致的调用方式，API 地址由各自部署环境决定：

```javascript
const API_BASE = '';  // 保持相对路径，两边都指向各自的 /api/chat
const API_URL  = '/api/chat';
```

#### (2) `index.html` — JS 模块路径（可选）

如需区分两套环境，可在 `index.html` 中引入不同配置。

## 四、新项目结构设计

### 4.1 目录结构

> ✅ **已简化**：新项目在 `cf-site/` 目录下，使用一体化 Worker，同时处理静态文件和 AI 接口。

```
cf-site/
├── worker/index.js         # Worker 主入口（一体化：静态服务 + /api/chat）
├── build.js                # 构建脚本（复制静态资源 + 替换 API URL）
├── wrangler.toml           # Cloudflare Workers 配置
├── package.json            # npm 依赖
├── .dev.vars               # 本地环境变量（需填入真实密钥）
└── dist/                   # 构建产物（由 build.js 自动生成）
    ├── index.html
    ├── css/
    ├── js/ai-panel.js      # API_URL 已替换为 '/api/chat'
    └── ...
```

### 4.2 API 层实现

一体化 Worker `worker/index.js` 同时提供两类功能：

| 路由 | 处理方式 |
|------|---------|
| `GET /api/chat` | CORS 预检（OPTIONS） |
| `POST /api/chat` | 流式转发到 LLM API（SSE 透传） |
| `GET /*`（其他路径） | 返回 `dist/` 下的静态文件 |

**核心优势**：无需额外的静态托管服务，Worker 自己处理文件，部署最简。

## 八、注意事项

### 8.1 数据一致性

- 两套系统共享同一个 Supabase 数据库
- 任何一边的数据变更会实时同步到另一边
- 无需额外的数据同步机制

### 8.2 API 兼容性

- Cloudflare Workers 有 100ms CPU 时间限制（流式输出会中断连接）
- 如遇超时问题，可使用 Cloudflare Durable Objects 或分段请求
- 当前 AI 对话逻辑为纯流式，需监控实际运行情况

### 8.3 域名与备案

- 使用 `.workers.dev` 域名：**无需备案**
- 如需绑定自定义域名（如 `cn.example.com`）：**需要备案**
- 建议先用 `.workers.dev` 测试，正式使用再考虑域名

### 8.4 成本说明

| 项目 | 免费额度 | 说明 |
|------|---------|------|
| Cloudflare Workers | 每天 10 万次请求 | 个人使用足够 |
| 带宽 | 无限 | Cloudflare 全球 CDN |
| 存储 | 有限 | Workers KV 小数据可用 |

## 十、总结

本计划利用 Cloudflare Workers 搭建国内访问节点，核心思路：

1. **不改动现有海外版** — 保持稳定运行
2. **复用现有代码** — 90% 以上代码直接复用
3. **共享数据库** — Supabase 作为统一数据层，两边完全同步
4. **最小改动** — 只需改动 AI 对话接口的框架适配代码

预计工作量：约 2-3 小时完成部署和基础测试。
